## 🎯 项目设计动机

本项目的设计初衷来源于当前国际急救培训方式的变化趋势。

随着急救培训逐渐从传统线下课堂，向“混合式教学（Blended Learning）”模式发展，国际组织与专业机构均在推动更高效、更可扩展的培训体系。

例如：

- 红十字与红新月体系 （IFRC）在相关培训方向中强调混合式教学与情景模拟训练的结合  
- 美国心脏协会 （AHA) 在其 BLS/CPR 培训体系中持续推广“线上学习 + 线下实操训练”的混合模式  
- 欧洲复苏委员会 (ERC) 在其急救复苏指南与培训框架中，同样支持线上学习、线下技能训练与模拟情景训练相结合的教学模式  

在此背景下，本项目尝试利用 VR 与 VRChat 平台构建一个用于：

- 情景化急救训练  
- AED 操作流程模拟

## 🫀 AED设备参考与交互逻辑说明

### 📌 设备来源说明

本项目最初在开发过程中，曾使用以下真实设备外观作为建模参考：

- **参考设备：Cardiolife AED-3100**
- **类型：半自动体外除颤器（AED）**

由于版权与分发规范原因，项目公开版本中已移除相关商业模型资源，仅保留自主修改与交互逻辑实现部分。

---

### ⚙️ 真实设备操作逻辑（参考）

Cardiolife AED-3100 的实际操作逻辑如下：

- 通过开启盖体（或解锁结构）自动激活设备电源  
- 部分版本无需独立电源按钮  
- 关闭盖体即关闭设备并终止运行状态  

该设计属于“开盖即启动”式 AED 结构。

---

### 🧩 VR交互实现逻辑（本项目）

在 VRChat 实现中，为适配教学与交互体验，系统采用如下模拟方式：

---

#### 🟢 开盖（启动）

- 玩家扳下 / 打开盖体卡扣  
- 触发“开盖事件”  
- 系统自动进入开机状态  
- 播放开机 / 开盖提示音  

对应真实设备：开盖即启动

---

#### 🟡 使用状态（分析流程）

- 玩家触发“分析”按钮 / 交互点  
- 系统进入心律分析状态  
- 若存在前置语音（如开盖提示音），将自动中断  
- 播放分析语音提示  

---

#### 🔴 关闭状态（收纳）

- 玩家关闭盖体卡扣  
- 系统立即停止所有音频与交互状态  
- 重置 AED 状态机  
- 模拟设备断电 / 待机状态  

对应真实设备：盖体关闭即关机
